ガイドにモデルカスタマイズがREST APIとVision Studioでのみ可能で、クライアント言語SDKではサポートされていないことが明示されました。これにより、ユーザーは使えるツールに関する明確な理解を持てるようになります。また、見出しやREST APIの利用に関する説明が改善され、ドキュメントの可読性が向上しています。

記事: articles/ai-services/disable-local-auth.md

ローカル認証の無効化に関するガイドが詳細化されました。具体的には、Microsoft Entra認証の強制方法、BicepやARMテンプレートの設定方法、PowerShellを用いた無効化手順が具体例とともに追加されました。また、ローカル認証のステータスを確認する方法も追加され、ユーザーが手順を正しく実施しているかを確認しやすくなっています。これにより、セキュリティ強化と管理の効率化が図れます。

これらの変更は全体的に、Azure AIサービスの利用者にとって情報の明確さと操作の利便性を高める目的で行われています。特に、セキュリティ面での強化や特定の機能利用に関する注意書きの追加は、ユーザーがサービスを正しくかつ効果的に活用する助けとなるでしょう。
